# 교회문서키트 BASIC 1.28

## 1.26 변경 사항

모바일에서 Supabase 이메일 Sign in 링크가 안정적으로 열리지 않는 문제를 피하기 위해 로그인 방식을 변경했습니다.

- 기존: 이메일 입력 → Supabase 로그인 메일 → Sign in 링크 클릭
- 변경: 승인된 이메일 입력 → 접속코드 입력 → 바로 접속

이제 베타테스터는 PC와 모바일에서 같은 방식으로 접속할 수 있습니다.

## 필수 Vercel 환경변수

기존 환경변수에 아래 값을 추가해 주세요.

```text
BETA_ACCESS_CODE=관리자가 정한 베타 접속코드
```

예시:

```text
BETA_ACCESS_CODE=crossdoc2026
```

기존 환경변수도 유지해야 합니다.

```text
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
SUPABASE_SERVICE_ROLE_KEY
ADMIN_PASSCODE
BETA_ACCESS_CODE
```

`APP_SESSION_SECRET`은 선택입니다. 없으면 `ADMIN_PASSCODE`를 세션 서명용으로 함께 사용합니다.

## 적용 방법

1. 압축을 풉니다.
2. GitHub에 압축 푼 파일 전체를 덮어쓰기 합니다.
3. Vercel 환경변수에 `BETA_ACCESS_CODE`를 추가합니다.
4. Vercel에서 Redeploy 합니다.
5. 승인된 이메일 + 접속코드로 PC/모바일 접속을 확인합니다.

## 기존 베타테스터 안내

기존에 받은 Supabase Sign in 메일은 사용하지 않아도 됩니다. 
앞으로는 작성기 주소에서 승인된 이메일과 접속코드를 입력해 주세요.

## 내 문서 저장

PC·모바일 이어쓰기 기능을 쓰려면 Supabase SQL Editor에서 `supabase_cloud_documents.sql` 내용을 한 번 실행해야 합니다.


## BASIC 1.28 모바일 간편 사용 보강

- PC 화면 구조는 유지하고, 900px 이하 모바일 화면만 별도 흐름으로 정리했습니다.
- 모바일 작성 화면은 `홈 / 입력 / 확인 / 저장` 흐름으로 이동합니다.
- 하단 고정바를 `홈 · 입력 · 확인 · 저장`으로 단순화했습니다.
- 저장 단계에서 `PC·모바일 이어쓰기`와 PDF/PNG 저장을 한곳에서 확인하도록 정리했습니다.
- 모바일 홈 대시보드는 문서 카드와 내 문서 목록이 더 크게 보이도록 보강했습니다.


## BASIC 1.28 모바일 동반모드

모바일에서 모든 서식을 완성하려고 하면 입력칸과 표가 많아 사용 부담이 큽니다. 1.27에서는 PC 편집 화면은 유지하고, 모바일은 “완성 편집기”가 아니라 “현장 초안 저장용”으로 정리했습니다.

- 모바일 첫 작성 영역을 제목/기간/장소/대상/핵심 메모 중심으로 단순화
- 복잡한 표·일정·예산은 기본적으로 PC에서 이어서 작성하도록 안내
- 모바일 간편 모드에서는 기존 빠른작성 탭을 숨기고, 필요할 때만 세부 서식을 열도록 변경
- PC 화면과 기존 문서 구조는 유지

운영 안내 문구:
모바일에서는 제목, 날짜, 장소, 핵심 메모만 먼저 저장해 주세요. 자세한 표와 PDF 출력은 PC에서 이어서 정리하면 가장 안정적입니다.


## BASIC 1.28 변경사항

- 교육부/부서별 주간보고서의 `출석/참여` 입력칸을 숫자 전용에서 자유 입력으로 변경했습니다.
- 이제 `24`, `학생 24명 / 교사 6명`, `현장 31명 · 온라인 3명`, `가정예배 5가정`처럼 글자와 숫자를 함께 입력할 수 있습니다.
- 숫자만 입력하면 기존처럼 큰 숫자+명 형태로 보이고, 글자가 포함되면 입력한 문구 전체가 표와 미리보기에 표시됩니다.
- 부서 통합 주간보고서의 부서별 현황 표에도 동일하게 반영했습니다.
